# Аудит Stain (backend + frontend) и план улучшений

Краткий обзор текущей реализации и направлений развития без привязки к конкретным спринтам.

## Backend

- **Безопасность и сессии**: JWT + cookie; стоит явно задокументировать срок жизни, обновление токена и политику `SameSite`/`Secure` для production.
- **Загрузки**: раздельные лимиты для изображений и видео (`UPLOAD_MAX_IMAGE_BYTES`, `UPLOAD_MAX_VIDEO_BYTES`); при деплое синхронизировать с лимитами PHP (`upload_max_filesize`, `post_max_size`); на VPS — ещё `client_max_body_size` в nginx; на shared — настройки в панели хостинга / `.user.ini`.
- **Медиа**: файлы на диске + записи в БД; удаление из медиатеки блокируется, если `/media/{id}` ещё есть в HTML поста/страницы (ответ API `409`, код `MEDIA_IN_USE`).
- **Маршрутизация**: диспетчеризация вынесена в `Http\Router` + трейт `AppRouteRegistration`; при росте проекта можно дробить регистрацию по файлам.
- **Валидация**: часть правил в сервисах, часть в обработчиках; для API постепенно использовать поле `code` (например `MEDIA_IN_USE`, `FORBIDDEN`) рядом с `error`.
- **Тесты**: добавить интеграционные тесты для постов, категорий, медиа и панели (минимальный happy-path + ошибки валидации).

## Frontend

- **Формы редактора**: контент в `contenteditable` + скрытые поля; клиентская валидация перед отправкой снижает потерю данных. Черновик в `localStorage` реализован (`draft-post.js`, см. таблицу «Желательно»).
- **Панель**: переключение разделов через редиректы; можно заменить на единый layout с подгрузкой секций без полной перезагрузки (по желанию).
- **Доступность**: попапы на `:target` и оверлеи — проверить фокус-ловушку и закрытие по Escape там, где это критично.
- **Производительность**: ленивая загрузка изображений, при необходимости — responsive `srcset` для обложек и вставок.

## UX / контент

- **Категории**: slug в URL как архив; убедиться, что SEO-метаданные и canonical на страницах категорий заданы осознанно.
- **Медиатека**: сетка в стиле photostream; при большом числе файлов — пагинация или виртуализация.

## Терминология (требование 8.1)

- В классах CSS, id и подписях интерфейса избегать слова **admin**; роль `admin` в данных и опциях ролей остаётся.
- Сообщения об ошибках в API по возможности нейтральные (`Forbidden`), без упоминания роли в тексте.

## Приоритетный бэклог (пример)

2. Проверка ссылок на медиа при удалении или предупреждение в UI.

## Что требуется сделать (по плану выше)

Ниже — практический порядок: сначала то, что критично для продакшена и безопасности, затем качество и скорость разработки.

### Критично перед production

| Задача | Суть |
|--------|------|
| Почта | `MAIL_FROM`, `ADMIN_EMAILS`; на shared-хостинге — почта из панели или SMTP; на VPS — postfix/sendmail или внешний SMTP. |
| Загрузки | Согласовать `UPLOAD_MAX_*` с лимитами PHP и веб-сервера (см. README «Развёртывание»). |
| Куки / JWT | Сделано в коде: переменные `AUTH_COOKIE_SECURE`, `AUTH_COOKIE_SAMESITE`, опционально `AUTH_COOKIE_DOMAIN`; срок жизни cookie = `JWT_TTL_SECONDS` (как у токена). В production задать `AUTH_COOKIE_SECURE=1`, при необходимости `Strict`; длинный уникальный `JWT_SECRET`. |

### Важно (надёжность и данные)

| Задача | Суть |
|--------|------|
| Удаление медиа | **Сделано:** блокировка удаления при использовании в HTML; фронт показывает список материалов. |
| Единый формат ошибок API | **Сделано в базовом виде:** `ApiErrorMapper` — ответы `{ error, code }` (и `references` для `MEDIA_IN_USE`); HTTP 401/403/404/409/422 где уместно. |

### Желательно (удобство и сопровождение)

| Задача | Суть |
|--------|------|
| Черновик в `localStorage` | **Сделано:** `draft-post.js` для форм создания/редактирования поста (`data-post-draft-form`), очистка при успешной отправке. |
| Роутер | **Сделано:** `Http\Router`, трейт `AppRouteRegistration`. |
| SEO по категориям | **Сделано:** мета, крошки, вводный текст в `category.php`; sitemap/robots/OG — см. README. |
| `srcset` / lazy | Публичный вывод постов/страниц: `loading="lazy"` на `<img>` через `HtmlPublicContent`; отдельный `srcset` под макет — по желанию. |

### Качество кода

| Задача | Суть |
|--------|------|
| Интеграционные тесты | API: посты, категории, медиа, панель; минимум happy-path + 422/401. |
| Smoke UI | Критические сценарии в браузере (логин, создание поста, загрузка файла). |

### Уже учтено в проекте (справочно)

- Раздельные лимиты изображений/видео в конфиге и `MediaService`.
- Клиентская валидация формы поста перед отправкой.
- Нейтральные тексты части ошибок API; панель без префикса `admin` в классах.
- Обратная связь с `MAIL_FROM` и `Reply-To` на email пользователя.
- JWT cookie: `AUTH_COOKIE_*` + срок cookie по `JWT_TTL_SECONDS` (см. `.env.example`).
- Удаление медиа с проверкой ссылок в постах/страницах; ответ 409 с `references`.

Когда брать задачу из списка **«Желательно»** — после стабилизации деплоя и мониторинга ошибок на production.
