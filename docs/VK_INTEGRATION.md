# Интеграция с VK (обзор для Stain)

Официальная документация: [dev.vk.com](https://dev.vk.com/ru/guide), [методы API](https://dev.vk.com/ru/method), [VK ID](https://id.vk.com/about/business/go/docs/ru/vkid/latest/vk-id/intro/start-page).

## Авторизация «Через VK» на сайте

Современный путь — **VK ID** (OAuth 2.0 / OpenID Connect), а не «легаси» standalone-приложения с `display=page`:

1. Зарегистрировать приложение в [кабинете разработчика](https://vk.com/apps?act=manage), указать **доверенный redirect URL** вашего сайта (например `https://example.com/auth/vk/callback`).
2. Получить **client_id** и **защищённый client_secret** (хранить только на сервере).
3. Поток: редирект пользователя на авторизацию VK ID → обмен `code` на **access_token** на бэкенде → получить **идентификатор пользователя VK** и при необходимости **email** (если запрашивали scope и разрешено политикой VK).
4. В **Stain** связать VK user id (и опционально email) с локальным пользователем в таблице `users` (новое поле `vk_id` или отдельная таблица связей) и выдать свой **JWT** так же, как при обычном логине (`setWebAuthCookie`).

Важно: почта и телефон даются не всегда — проект должен иметь сценарий «профиль без email» или дозапроса.

## «Комментарии в проекте» и API ВК

Разделить три разных сценария:

### A. Комментарии **на сайте Stain** (своя БД)

- VK API **не хранит** комментарии к вашим URL. Нужны свои таблицы (`comments` с `post_id`, `user_id`/`vk_user_id`, текст, дата).
- Авторизация комментатора: ваш JWT после логина (в т.ч. через VK ID) или гостевой режим с модерацией.

### B. Виджет комментариев VK на странице

- В [документации API](https://dev.vk.com/ru/method) есть **widgets.getComments** — выборка комментариев, оставленных через **виджет комментариев VK** к привязанным страницам сайта.
- Требуется связка сайта/приложения с виджетом, обсуждение **не в вашей PostgreSQL**, а у VK; оформление — через их скрипт/виджет на шаблоне поста.

### C. Комментарии к **записи на стене VK**

- Методы **`wall.createComment`**, **`wall.getComments`** (раздел Wall) работают с **записью VK** (owner_id, post_id), а не с HTML-лентой блога на своём домене.
- Подходит, если вы кросс-постите пост в группу и хотите обсуждение там; для чистого блога на shared-хостинге это отдельный продуктовый контур.

## Практическая рекомендация для Stain на shared-хостинге

1. **Вход через VK ID** — реализуемый слой: endpoints `GET /auth/vk`, `GET /auth/vk/callback` в PHP, без Docker-специфики.
2. **Комментарии под постами** — хранить у себя в БД; при желании добавить **опционально** виджет VK на `templates/post.php` параллельно.

Ограничения API (лимиты, версия `v=…`, устаревшие методы) — всегда сверять с актуальным [справочником](https://dev.vk.com/ru/method).
